# OBS 映像エフェクトプラグイン 設計仕様書

> このドキュメントは Claude Code への実装引き継ぎを目的とした設計仕様です。
> 「要件」「実装手段（技術的アプローチ）」「制御パラメーター（UX観点で設計）」を各エフェクトごとに定義します。
> 仕様が確定していない箇所には **【要確認】** を付けています。実装着手前に発注者へ確認してください。

## 0. 全体方針

### 0.1 ゴール
OBS Studio 上で、映像ソース全体に対して以下10種のエフェクトをリアルタイムに適用できるプラグインを作る。各エフェクトは独立しており、ユーザーが任意に組み合わせ（スタック）できる。

対象エフェクト：カートゥーン / セピア / 古い映画 / VHS / ドット絵 / ブルーム / フォーカス（単焦点風）/ ガビガビ（低画質圧縮）/ インスタ風色補正 / グリッチ

### 0.2 アーキテクチャ決定

| 論点 | 決定 | 理由 |
|---|---|---|
| プラグイン形態 | **ネイティブ C/C++ プラグイン**（`obs-plugintemplate` ベース、CMake） | リアルタイム GPU 処理に必要なパフォーマンスと、複数パス描画の自由度が得られる。`obsproject/obs-plugintemplate` が公式の出発点。 |
| エフェクトの単位 | **1エフェクト = 1フィルター**（`obs_source_info`, `OBS_SOURCE_TYPE_FILTER \| OBS_SOURCE_VIDEO`）。全フィルターを1つのプラグインモジュールに同梱 | OBS のフィルターは右クリック→フィルターで個別に追加・順序入替・スタックできる。「ドット絵＋グリッチ」のような重ねがけが自然に実現でき、OBS の作法に沿う。 |
| 描画 | OBS の **エフェクトシステム（`.effect` ファイル / OBS版HLSL）** を使用。GPUシェーダで処理 | OBS版HLSLは D3D/OpenGL（Metal）へクロスコンパイルされる。CPU処理は避ける。 |
| 単一パスのエフェクト | `obs_source_process_filter_begin()` → `gs_technique` 描画 → `obs_source_process_filter_end()` | セピア・カートゥーン等、入力を1回サンプルするだけのものはこの最短経路で実装。 |
| 複数パスのエフェクト | **`gs_texrender_t`** で中間レンダーターゲットを確保し、ダウンサンプル→ブラー→合成 | ブルーム・フォーカス・グリッチのブルーム成分・ガビガビ等に必須。 |
| 時間アニメーション | エフェクトパラメータ `elapsed_time`（OBS提供）または `video_tick` コールバックで経過時間を渡す | 古い映画の縦線移動、VHS/グリッチのランダム発生に使用。 |
| パラメータ保存 | OBS の `obs_data_t` 設定に自動シリアライズ。`get_defaults` で既定値、`get_properties` でUI定義 | OBS標準。シーンコレクションに保存される。 |

> **プロトタイピングの近道**：シェーダ単体の試作は既存の `obs-shaderfilter`（exeldro 版）に `.shader` を読み込ませて画作りを詰め、確定したらネイティブプラグインへ移植すると速い。最終成果物はネイティブプラグインとする。

### 0.3 クロスプラットフォーム上の注意（シェーダ）
- OBS版HLSLで書き、`image` を入力テクスチャ、`uv_size`（テクスチャ解像度）等の標準uniformを利用する。
- OpenGL（Linux）向けクロスコンパイルは万能ではないため、HLSL構文はシンプルに保つ（複雑な分岐・配列初期化子を避ける）。`float4 px = image.Sample(...)` の素直な書き方を徹底。
- 解像度依存のエフェクト（ブルーム半径、ピクセルサイズ等）は**ピクセル単位ではなくUV/解像度比で正規化**し、1080p/4Kで見た目が破綻しないようにする。

### 0.4 UX 共通ルール（全エフェクト必須）
1. **「強度 / ミックス」スライダー（0–100%、既定値はエフェクトごとに調整）を全エフェクトに必ず用意**する。元映像とのブレンド量を最後に lerp で掛け、ユーザーが効果を弱められるようにする。「効きすぎ」は最大の不満要因。
2. **既定値は単体追加した瞬間に“それっぽく見える”値**にする（無設定で破綻しない）。
3. パラメータは **「基本」と「詳細」にグルーピング**（`obs_properties_add_group`）。初見ユーザーは基本だけ触れば十分にする。
4. ウィジェット対応：連続値→スライダー（`*_slider`）、離散選択→ドロップダウン（`obs_properties_add_list`）、色→カラーピッカー（`obs_properties_add_color`）、ファイル→`obs_properties_add_path`。
5. 重い処理（ブルーム・フォーカス・ガビガビ）には **「品質」段階（低/中/高）** を用意し、低スペック環境で落とせるようにする。
6. ラベルは**エンドユーザー向け日本語**で命名（内部キーは英語）。数値の単位（px / % / 度）をラベルに明記。

## 1. カートゥーン（Cartoon）

### 要件
映像をアニメ／セル画調にスタイライズする。元要件には「映像のひずみ」とあるが、一般的なカートゥーン化は **輪郭線の付与 + 色の階調化（ポスタリゼーション）+ 平滑化** を指すことが多い。本仕様ではこれを主機能とし、任意の軽い歪みをオプション扱いにする。

> **【要確認】**「映像のひずみ」が *セル画調（色数削減＋輪郭線）* を指すのか、*レンズ歪み/波打ちのような変形* を指すのかを発注者に確認。本仕様は前者で設計。

### 実装手段
1. **輪郭抽出**：Sobel もしくは差分エッジ検出で輝度勾配を取り、しきい値以上を黒線として上から重ねる。線の太さはサンプル間隔で制御。
2. **色の階調化（ポスタライズ）**：`floor(color * levels) / levels` で各チャンネルを N 段階に量子化し、ベタ塗り感を出す。
3. **平滑化（任意）**：軽いバイラテラル/ボックスブラーを量子化前にかけ、ディテールを潰してアニメ的な面を作る。
4. **彩度ブースト（任意）**：HSV で彩度を持ち上げてポップにする。

| ラベル（日本語） | 内部キー | 型 | 範囲 | 既定値 | ウィジェット | グループ |
|---|---|---|---|---|---|---|
| 強度 | strength | float | 0–100% | 100 | スライダー | 基本 |
| 輪郭線の強さ | edge_strength | float | 0–100% | 60 | スライダー | 基本 |
| 輪郭線の太さ | edge_width | float | 0.5–4.0 px | 1.5 | スライダー | 基本 |
| 色の階調数 | color_levels | int | 2–32 | 8 | スライダー | 基本 |
| 平滑化 | smoothing | float | 0–100% | 40 | スライダー | 詳細 |

## 2. セピア（Sepia）

### 要件
映像をセピア調（褐色のモノトーン）にする。単体で完結する軽量エフェクト。

### 実装手段
1. 輝度を計算（`dot(rgb, float3(0.299,0.587,0.114))`）。
2. 輝度に対し**セピア基準色**を掛けて褐色化（標準セピア変換行列、または「影色→ハイライト色」のグラデーションマップ）。
3. `mix(original, sepia, strength)` でブレンド。コントラストとビネットを任意で付与。

## 3. 古い映画（Old Film）

### 要件
セピア調をベースに、古いフィルム映写のノイズを再現する：
- フィルムグレイン（粒状ノイズ）
- **縦方向の黒い線（キズ）が、ゆっくり左右へ流れる**
- 黒い点・斑点（ホコリ）が時々入る
- （映写機の）明滅・揺れ

### 実装手段
1. **ベース**：セピア（#2）または脱色＋トーン。
2. **グレイン**：時間シード付き乱数ノイズを輝度に加算（毎フレーム更新で“ざらつき”）。
3. **縦キズ**：複数本の細い縦線を手続き的に生成。各線の X 位置を `base_x + sin(time * speed + seed)` 等でゆっくり左右へ移動させる。線は暗く（黒寄り）に乗算。本数・速さ・濃さをパラメータ化。
4. **ホコリ/斑点**：時間シードで毎フレームごく短時間だけ出現する小さな暗/明スポットをランダム配置（出現確率＝量パラメータ）。
5. **明滅（フリッカー）**：全体輝度を `1 + flicker * noise(time)` で微小に揺らす。
6. **ビネット/フレーム揺れ（任意）**：周辺減光と、わずかな上下左右オフセットで映写の不安定さを演出。

> 時間入力は `elapsed_time` を使用。乱数は `frac(sin(dot(uv, k)) * m)` 系のハッシュで GPU 上生成し、フレームごとに `time` をシードに混ぜる。

| 強度 | strength | float | 0–100% | 100 | スライダー | 基本 |
| セピア量 | sepia_amount | float | 0–100% | 80 | スライダー | 基本 |
| グレイン量 | grain | float | 0–100% | 45 | スライダー | 基本 |
| 明滅の強さ | flicker | float | 0–100% | 30 | スライダー | 基本 |
| 縦キズの本数 | scratch_count | int | 0–12 | 3 | スライダー | 基本 |
| 縦キズの流れる速さ | scratch_speed | float | 0–100% | 25 | スライダー | 詳細 |
| 縦キズの濃さ | scratch_intensity | float | 0–100% | 60 | スライダー | 詳細 |

## 4. VHS

### 要件
ビデオテープ風の劣化：ノイズ、色のにじみ、走査線（操作線）、時々グリッチ的な歪み。

> **【要確認】**「操作線」は走査線（スキャンライン）／トラッキングノイズ帯のいずれか、または両方の意。本仕様では両方を別パラメータで用意。

### 実装手段
1. **色のにじみ（クロマブリード）**：R/G/B（特にクロマ）成分を水平方向に少しずつオフセットしてサンプルし、色を横へ引きずる。輝度はそのまま、色だけ滲ませるのが本物っぽい。
2. **ノイズ**：輝度に時間シードノイズを加算（テープノイズ）。
3. **走査線**：`sin(uv.y * lineCount)` で水平な暗線パターンを乗算。
4. **トラッキングノイズ（操作線）**：画面を上下にゆっくり移動する**横帯**を生成し、その帯内で強いノイズ・水平ずれを発生。帯の位置を `frac(time * speed)` で動かす。
5. **グリッチ歪み**：時間トリガで時々、水平スライスを左右にずらす（発生頻度・強度をパラメータ化）。
6. （任意）全体の彩度低下・エッジゴースト。

|---|---|---|---|---|---|---|
| 強度 | strength | float | 0–100% | 100 | スライダー | 基本 |
| 色のにじみ | chroma_bleed | float | 0–100% | 50 | スライダー | 基本 |
| ノイズ量 | noise | float | 0–100% | 40 | スライダー | 基本 |
| 走査線の強さ | scanline | float | 0–100% | 35 | スライダー | 基本 |
| トラッキングノイズ量 | tracking_amount | float | 0–100% | 30 | スライダー | 基本 |
| トラッキング移動速度 | tracking_speed | float | 0–100% | 20 | スライダー | 詳細 |
| グリッチ発生頻度 | glitch_freq | float | 0–100% | 15 | スライダー | 基本 |
| グリッチ強度 | glitch_intensity | float | 0–100% | 40 | スライダー | 詳細 |

## 5. ドット絵（Pixel Art）

### 要件
- ピクセルサイズを指定してドット風（モザイク化）にする。
- 色数を **16色 / 256色** から選べる。
- **16色**の場合、パレットから閾値を外れた色は「昔ながらの2色で埋める方法」＝**ディザリング**で表現する。

### 実装手段
1. **ピクセル化**：UV を `floor(uv / cell) * cell + cell*0.5` にスナップしてセル中心をサンプルし、指定ピクセルサイズのブロックにする。
2. **色量子化**：
   - **256色**：固定パレット（または各チャンネルを段階量子化、例 R3:G3:B2bit 相当）にマッピング。
   - **16色**：選択した**16色固定パレット**へ最近傍マッピング。
3. **16色時のディザリング**：各ピクセルについて、元色とパレット最近傍色の**距離（誤差）が閾値を超えた場合**、近い2色を Bayer マトリクス（4×4 等の順序ディザ）で交互配置して中間色を擬似表現する（＝「2色で埋める」）。閾値以内なら最近傍色をそのまま使う。
4. パレットは**プリセット選択式**にする（リアルタイムで動的パレット生成はコスト高のため固定推奨）。

> パレット候補例：クラシック16（EGA系）／ゲームボーイ4階調（緑）／PC-98風／モノクロ2色 など。`obs_properties_add_list` でドロップダウン化。

### パラメーター
| ラベル | 内部キー | 型 | 範囲/選択肢 | 既定値 | ウィジェット | グループ |
|---|---|---|---|---|---|---|
| 強度 | strength | float | 0–100% | 100 | スライダー | 基本 |
| ピクセルサイズ | pixel_size | int | 1–64 px | 8 | スライダー | 基本 |
| 色数 | color_count | enum | 16 / 256 / フルカラー | 256 | ドロップダウン | 基本 |
| パレット | palette | enum | クラシック16/ゲームボーイ/EGA/モノクロ ほか | クラシック16 | ドロップダウン | 基本（16色時のみ有効） |
| ディザリング閾値 | dither_threshold | float | 0–100% | 35 | スライダー | 基本（16色時のみ有効） |
| ディザの強さ | dither_strength | float | 0–100% | 80 | スライダー | 詳細（16色時のみ有効） |

> **UX**：色数が「16」のときだけパレット／ディザ系を有効表示する（`obs_property_set_visible` を `update` で切替）。256色／フルカラー時は不要項目を隠して迷わせない。

## 6. ブルーム（Bloom）

### 要件
妥当な手法を調査して実装。**明るい部分にだけ**グローを発生させたい。

### 実装手段（調査結果に基づく標準パイプライン）
OBS は基本 LDR（8bit）描画なので、HDR前提ではなく **しきい値ベースの輝度抽出（ブライトパス）** が適切。3段構成：

1. **ブライトパス（明部抽出）**：輝度 `luma` を計算し、`bloom = smoothstep(threshold, threshold + knee, luma)` で**しきい値以上だけ**を抽出（ソフトニーで急な切れ目を回避）。これにより「明るい部分にだけ」を満たす。
2. **ダウンサンプル＋ブラー**：抽出画像を 1/2・1/4…と段階的に縮小し、各段で小カーネル（3×3〜5×5）ブラーをかける。`gs_texrender_t` を複数用意してピンポン描画。縮小＋ガウシアン近似で広範囲の拡散を低コストに得る（LearnOpenGL の物理ベース手法の縮小・拡大ピラミッドが参考）。
3. **アップサンプル＋加算合成**：縮小バッファを順に拡大しながら足し戻し、最終的に元映像へ **加算（additive）** で合成。合成量＝強度パラメータ。

> 拡散半径を**解像度比で正規化**し、1080p/4Kで効果が消えない／効きすぎないようにする。品質パラメータでブラー反復回数・段数を切替。

### パラメーター
| ラベル | 内部キー | 型 | 範囲 | 既定値 | ウィジェット | グループ |
|---|---|---|---|---|---|---|
| 強度 | intensity | float | 0–200% | 80 | スライダー | 基本 |
| しきい値（明るさ） | threshold | float | 0–100% | 70 | スライダー | 基本 |
| ソフトさ（ニー） | knee | float | 0–100% | 50 | スライダー | 基本 |
| 拡散の広さ | radius | float | 0–100% | 50 | スライダー | 基本 |
| 色付け（ティント） | tint | color | — | 白(無着色) | カラーピッカー | 詳細 |
| 品質 | quality | enum | 低 / 中 / 高 | 中 | ドロップダウン | 詳細 |

## 7. フォーカス（単焦点風 / Focus）

### 要件
起点から外へ向かうにつれてぼかしが強くなる（被写界深度風）。**強度のカーブを調整**できる。

### 実装手段
1. **マスク生成**：フォーカス起点（中心 X/Y）からの距離 `d` を計算。形状（円/楕円）と縦横比を考慮。内側の「ピントが合う半径」と「移行幅」で正規化：`t = saturate((d - innerRadius) / falloff)`。
2. **カーブ適用**：`blurAmount = pow(t, curve)` 等の指数でぼけの立ち上がりカーブを制御（curve<1で中心付近から早くぼける、curve>1で外周だけ強くぼける）。**カーブを可変にするのが本要件の肝**。
3. **可変ブラー**：あらかじめ全面をブラーした版を `gs_texrender` で用意し、`mix(sharp, blurred, blurAmount)` で合成（簡易・高速）。より自然にしたい場合は複数強度のブラー段を作って多段 lerp。
4. 中心位置はプレビュー上で直感的に置けるよう X/Y スライダー＋（可能なら）ガイド表示。

### パラメーター
| ラベル | 内部キー | 型 | 範囲 | 既定値 | ウィジェット | グループ |
|---|---|---|---|---|---|---|
| 強度 | strength | float | 0–100% | 100 | スライダー | 基本 |
| 起点 X | center_x | float | 0–100% | 50 | スライダー | 基本 |
| 起点 Y | center_y | float | 0–100% | 50 | スライダー | 基本 |
| ピント範囲（内側半径） | inner_radius | float | 0–100% | 20 | スライダー | 基本 |
| 移行幅（ぼけ始め幅） | falloff | float | 1–100% | 40 | スライダー | 基本 |
| ぼけカーブ | curve | float | 0.2–5.0 | 1.5 | スライダー | 基本 |
| 最大ぼかし量 | max_blur | float | 0–100% | 70 | スライダー | 基本 |
| 形状 | shape | enum | 円 / 楕円 | 円 | ドロップダウン | 詳細 |
| 縦横比（楕円時） | aspect | float | 0.2–5.0 | 1.0 | スライダー | 詳細 |

## 8. ガビガビ（低画質・高圧縮 / Crunch）

### 要件
昔のインターネット動画のような、**低解像度でものすごく圧縮された映像**を再現する。

### 実装手段
古い動画＝**ブロックDCT圧縮＋クロマサブサンプリング＋色深度不足** の組み合わせとして近似：
1. **解像度低下**：一度低解像度へ縮小→拡大してブロックっぽさ・ボケを出す（`gs_texrender` で低解像度ターゲットに描画→拡大サンプル）。
2. **ブロック量子化（DCT風）**：8×8 程度のブロック単位で平均化／量子化を強めて、圧縮特有のブロックノイズを発生。圧縮強度でブロック内の階調を粗くする。
3. **クロマサブサンプリング**：YCbCr に変換し、色差成分（Cb/Cr）だけ水平・垂直に間引いてから戻すと、色が滲んで「色がブロックでズレる」古さが出る。
4. **色深度低下（バンディング）**：各チャンネルを量子化して階調段差（マッハバンド）を作る。
5. （任意）**ブロックノイズ／モスキートノイズ**：エッジ周辺にリンギングを付与。フレームレート低下感を出すなら間欠ホールドも可。

|---|---|---|---|---|---|---|
| 強度 | strength | float | 0–100% | 100 | スライダー | 基本 |
| 解像度低下 | downscale | float | 0–100% | 60 | スライダー | 基本 |
| 圧縮ブロックサイズ | block_size | int | 2–32 px | 8 | スライダー | 基本 |
| 圧縮強度 | compression | float | 0–100% | 70 | スライダー | 基本 |
| 色深度（階調） | color_depth | int | 2–8 bit | 5 | スライダー | 基本 |

## 9. インスタ参考の色補正（Instagram-style Grade）

### 要件
調査に基づくInstagram風の色補正。

### 調査メモ（実装方針の根拠）
「インスタ風」は単一の処理ではなく、複数の定番処理の組み合わせ。共通する要素は：
- **黒の持ち上げ（フェード/マット）**：シャドウを完全な黒にせず浮かせる→フィルム/レトロ感。
- **スプリットトーン**：シャドウを寒色（ティール）、ハイライト/中間（特に肌）を暖色（オレンジ）へ寄せる（いわゆるティール＆オレンジ）。肌が映え、被写体が背景から浮く。
- **トーンカーブ＋彩度調整**：自然な彩度（vibrance）優先で上げ、競合する色（緑・マゼンタ）を抑える。
- **色温度のわずかな暖色寄せ／ビネット／微粒子**。

### 実装手段（パラメトリック＋LUTのハイブリッド）
ワンクリックの満足度と微調整の自由度を両立させるため、以下を併用：
1. **プリセット**（ドロップダウン）：「ティール&オレンジ」「フェード（マット）」「暖色フィルム」「クール」「ヴィンテージ」等を内蔵し、内部で下記パラメータ群を一括設定。
2. **マスター強度**：プリセット適用量を一括で増減。
3. **パラメトリック微調整**：色温度／色合い／彩度／自然な彩度／コントラスト／フェード（黒持ち上げ）／ハイライト色／シャドウ色／ビネット／粒状感。
4. **LUT 読み込み（任意）**：`.cube` ファイルを `obs_properties_add_path` で選択し、3D LUT として適用（既存資産や配布LUTを使いたい上級者向け）。

> 内部実装はリフト/ガンマ/ゲイン（lift-gamma-gain）＋スプリットトーン＋トーンカーブで構成すると、ほとんどのインスタ系ルックを再現できる。

## 10. グリッチ（Glitch）

### 要件
乱れる頻度と強度、にじみ、ブルーム（明るいところだけ）、操作線（走査線）、カラーオーバーレイ。複数のグリッチ要素を統合した“画面の乱れ”。

### 実装手段
時間トリガで複数の破壊効果を確率的に発生させる統合エフェクト：
1. **発生トリガ**：`frequency` に応じて、時間ハッシュで「今フレーム乱れるか」を判定。乱れ強度は `intensity` でスケール。
2. **スライス変位**：画面を水平の帯に分割し、帯ごとに左右へランダムオフセット（デジタル破綻）。
3. **にじみ（RGB/クロマシフト）**：R/B チャンネルを水平にずらしてサンプルし色ズレを発生。
4. **ブルーム（明部のみ）**：#6 のブライトパス（しきい値）→ブラー→加算を流用。「明るいところだけ」を満たす。
5. **走査線（操作線）**：水平暗線パターンを乗算。
6. **カラーオーバーレイ**：指定色を画面全体に `screen`/`overlay`/加算等でブレンド（乱れの瞬間だけ強める運用も可）。

> ブルーム成分は #6 と共通モジュール化し、コードを再利用する。

### 11.2 各フィルターの最小実装テンプレート
`obs_source_info` に以下を実装：
- `get_name` / `create` / `destroy` / `update`（設定反映）
- `get_defaults`（既定値・§各表の既定値）
- `get_properties`（§各表のウィジェット。基本/詳細グループ分け。条件付き表示は `update` 内で `obs_property_set_visible`）
- `video_render`（描画。単一パスは `obs_source_process_filter_begin/end`、複数パスは `gs_texrender_t` 利用）
- 時間アニメ系は `video_tick` で経過時間を蓄積し eparam へ渡す

### 11.3 共通モジュール化すべきもの
- **ブライトパス（しきい値＋ソフトニー）**：#6 Bloom・#10 Glitch で共用
- **ダウンサンプル/ガウシアンブラー（ピンポン）**：#6・#7・#8・#10 で共用
- **時間ハッシュ乱数（GPUノイズ）**：#3・#4・#10 で共用
- **強度ブレンド（最終 lerp）**：全フィルター共通の後段

### 11.4 実装順序（簡単→複雑、依存関係考慮）
1. **セピア**（最小・単一パス。テンプレート確立）
2. **カートゥーン**（単一パス、エッジ＋ポスタライズ）
3. **ドット絵**（単一パス＋パレット・ディザ、条件付きUI）
4. **インスタ風色補正**（単一パス＋プリセット・LUT）
5. **共通ブラー/ブライトパス基盤を整備**
6. **ブルーム**（複数パスの基盤確認）
7. **フォーカス**（ブラー基盤の流用）
8. **ガビガビ**（低解像度ターゲット＋ブロック量子化）
9. **古い映画**（時間アニメ＝縦線移動・ホコリ）
10. **VHS**（時間アニメ＋トラッキング帯）
11. **グリッチ**（全部入り。ブルーム・走査線・オーバーレイ統合）

### 11.5 ビルド／配布
- `git clone --recursive https://github.com/obsproject/obs-plugintemplate.git` を出発点に CMake 構成。
- ビルド：`cmake -DCMAKE_BUILD_TYPE=Release ...` → `cmake --build`。
- 配布物：各OSのプラグインディレクトリへ `.dll`/`.so`/`.plugin` と `data/`（.effect・locale）を配置。
- README にインストール手順・対応OS・必要OBSバージョンを明記。

### 11.6 パフォーマンス指針
- 1080p/60fps を基準予算に。1エフェクトあたり概ね 1ms 以下を目標（複数スタック前提）。
- 複数パス系（ブルーム・フォーカス・ガビガビ）はブラーを縮小バッファで行い、`quality` で段数・反復を可変に。
- 4K では縮小率を上げてサンプル数を抑える（解像度比正規化を徹底）。

### 11.7 テスト観点
- 各エフェクト単体追加で、無設定（既定値）でも破綻せず“それらしく”見えるか。
- `強度 0%` で原映像に完全復帰するか（全エフェクト共通の受け入れ基準）。
- スタック（例：ドット絵→グリッチ）が破綻しないか。
- 1080p / 4K / 異なるアスペクト比で見た目が一貫するか。
- 時間アニメ系の乱数がフレームごとに更新され、かつ過剰にチカチカしないか。
- Windows(D3D) と Linux/macOS(OpenGL/Metal) でシェーダが同等に動くか。
